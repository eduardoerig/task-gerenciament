# Engenharia de Requisitos (Resumo)

## Requisitos funcionais
1. Usuário deve cadastrar e autenticar conta.
2. Usuário deve criar, editar, excluir e concluir tarefas de lazer.
3. Tarefas devem ser categorizadas e priorizadas.
4. Sistema deve exibir agenda e histórico de atividades.
5. Sistema deve gerar estatísticas por categoria e streak.
6. Sistema deve permitir metas semanais/mensais.
7. Sistema deve sugerir novas atividades com base no comportamento.
8. Sistema deve manter base de conquistas (gamificação).

## Requisitos não funcionais
- Interface responsiva, foco mobile-first.
- Acessibilidade (contraste, leitura, tamanho de fonte).
- API REST com validação de entrada e autenticação.
- Arquitetura modular e escalável.
- Segurança mínima: hash de senha e JWT.
